Celery Exporter is a Prometheus exporter that wraps Celery's built-in real-time monitoring API to expose task lifecycle events, worker status, and queue metrics. It works with both Redis and RabbitMQ brokers and deploys as a Docker container, standalone binary, or Python package.

## Why Celery Exporter?

Existing Celery exporters either exposed incorrect metric values or failed to capture the full range of task lifecycle events. Celery Exporter hooks directly into Celery's native real-time event processing to deliver accurate, low-latency metrics without polling.

## Get started

<Steps>
  <Step title="Enable Celery events">
    Configure your Celery app to emit task and worker events to the broker.

    ```python celeryconfig.py
    worker_send_task_events = True
    task_send_sent_event = True
    ```
  </Step>
  <Step title="Run the exporter">
    Start Celery Exporter pointing at your broker. The exporter listens on port `9808` by default.

    ```bash
    docker run -p 9808:9808 danihodovic/celery-exporter \
      --broker-url=redis://redis:6379/0
    ```
  </Step>
  <Step title="Scrape metrics">
    Add Celery Exporter as a Prometheus scrape target and verify metrics are available.

    ```bash
    curl http://localhost:9808/metrics
    ```
  </Step>
  <Step title="Import Grafana dashboards">
    Import the pre-built dashboards from Grafana.com to visualize task throughput, failure rates, and worker health.
  </Step>
</Steps>
